# Domain Docs

Layout: **single-context**

| Artifact | Path |
|---|---|
| Domain language | `CONTEXT.md` (repo root) |
| Architectural decisions | `docs/adr/` (repo root) |

## Consumer rules

1. Read `CONTEXT.md` before starting any architecture, design, or diagnosis task.
2. Check `docs/adr/` for decisions already documented before proposing a new approach.
3. If `CONTEXT.md` doesn't exist yet, create it once you have enough domain understanding.
4. New ADRs go in `docs/adr/NNNN-title.md`.
